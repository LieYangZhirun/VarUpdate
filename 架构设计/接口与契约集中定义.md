## 一、跨脚本共享契约

### 1.1 事件协议

#### 事件名常量

```typescript
/**
 * 跨脚本事件名常量
 * 所有事件均通过酒馆助手事件 API 传输
 */
const EVENTS = {
  // ─── Agents 广播 ───
  PIPELINE_STARTED:  'agents:pipeline_started',
  STEP_STARTED:      'agents:step_started',
  MESSAGE_COMPLETE:  'agents:message_complete',
  STEP_RETRY:        'agents:step_retry',
  STEP_FAILED:       'agents:step_failed',
  PIPELINE_ENDED:    'agents:pipeline_ended',

  // ─── VarUpdate 广播 ───
  VAR_INITIALIZED:   'varupdate:initialized',
  VAR_UPDATED:       'varupdate:updated',
  VAR_UPDATE_FAILED: 'varupdate:update_failed',
  VAR_SCHEMA_READY:  'varupdate:schema_ready',

  // ─── 跨脚本协作 ───
  RETRY_REQUESTED:   'varupdate:retry_requested',
} as const;
```

#### 事件总览

| 事件名 | 方向 | 触发时机 |
|--------|------|---------|
| `agents:pipeline_started` | Agents → 外部 | 管道开始执行 |
| `agents:step_started` | Agents → 外部 | 单个 Agent 步骤开始 |
| `agents:message_complete` | **Agents → VarUpdate** | 单步生成完成（所有写入方式均触发） |
| `agents:step_retry` | Agents → 外部 | 单步触发重试 |
| `agents:step_failed` | Agents → 外部 | 单步彻底失败 |
| `agents:pipeline_ended` | Agents → 外部 | 管道执行结束 |
| `varupdate:initialized` | VarUpdate → 外部 | 变量初始化完成 |
| `varupdate:updated` | **VarUpdate → Agents** | 变量更新成功（重试联动的成功信号） |
| `varupdate:update_failed` | **VarUpdate → Agents** | 变量更新失败（重试联动的失败信号） |
| `varupdate:schema_ready` | VarUpdate → 外部 | Schema 编译完成 |
| `varupdate:retry_requested` | **Agents → VarUpdate** | 通知 VarUpdate 回退变量状态 |

> **加粗**的行是两个脚本之间有直接依赖关系的关键事件。

**VarUpdate 与 Agents 的耦合边界**：VarUpdate 在脚本内部使用「容错阈值」等规则决定单次解析是否记为失败，并在负载中附带 `discardedCount` 等字段供排查。**Agents 只需**根据 `varupdate:updated`（视为本步变量侧通过，可继续流水线）与 `varupdate:update_failed`（视为失败，可按约定广播 `varupdate:retry_requested` 等）做二分决策，**无需**读取或与 VarUpdate 的全局「容错阈值」配置保持同步。

#### 核心交互时序

```text
用户点击发送
  │
  ├─ Agents 广播 PIPELINE_STARTED
  │
  ├─ 遍历启用的 Agent:
  │   │
  │   ├─ 广播 STEP_STARTED
  │   ├─ 组装提示词 → API 调用 → 后处理 → 写入
  │   ├─ 广播 MESSAGE_COMPLETE ──────→ VarUpdate 监听
  │   │                                      │
  │   │                        ┌──────────────┴────────────────────────────┐
  │   │                        ▼                                             ▼
  │   │              本条含 <Var_Update> 且联动判定完成              本条不含 <Var_Update>
  │   │              （可与 Initial / 无标签继承等组合）            （继承上一层 data 等）
  │   │                        │                                             │
  │   │              ┌─────────┴─────────┐                                   ▼
  │   │              ▼                   ▼                         不广播 varupdate:updated /
  │   │       广播 VAR_UPDATED    广播 VAR_UPDATE_FAILED           update_failed（Initial 成功
  │   │       （丢弃≤阈值等）     （解析失败/丢弃>阈值）           时另广播 VAR_INITIALIZED）
  │   │              │                   │
  │   │              ▼                   ▼
  │   │       Agents: 通过        Agents: 广播 RETRY_REQUESTED
  │   │                                  │
  │   │                           VarUpdate: 回退变量
  │   │                                  │
  │   │                           Agents: 重试本步骤
  │   │                           广播 STEP_RETRY
  │   │
  │   └─ 步骤完成 or 失败（广播 STEP_FAILED）
  │
  └─ 广播 PIPELINE_ENDED
```

#### 事件负载类型定义

```typescript
// ═══════════════════════════════════════════════════════════
//  Agents 广播的事件负载
// ═══════════════════════════════════════════════════════════

/** agents:pipeline_started */
interface PipelineStartedPayload {
  /** 当前方案名称 */
  schemeName: string;
  /** 启用的智能体数量 */
  agentCount: number;
}

/** agents:step_started */
interface StepStartedPayload {
  /** 智能体唯一 ID */
  agentId: string;
  /** 智能体标签名 */
  agentLabel: string;
  /** 步骤序号（从 0 开始） */
  stepIndex: number;
  /** 步骤字母标识（"A", "B", "C"...） */
  stepLetter: string;
}

/**
 * agents:message_complete
 *
 * 最关键的跨脚本事件。VarUpdate 通过此事件接收 Agents 流水线
 * 产生的消息内容，即使「仅解析」模式也会广播。
 */
interface MessageCompletePayload {
  /** 智能体唯一 ID */
  agentId: string;
  /** 智能体标签名 */
  agentLabel: string;
  /** 步骤序号 */
  stepIndex: number;
  /** 生成的完整消息文本（经后处理） */
  content: string;
  /** 推理/思维链内容（若配置传回） */
  reasoning?: string;
  /**
   * chat[] 中的消息索引。
   * - 写入聊天记录的模式：有值
   * - 「仅解析」模式：undefined
   */
  messageIndex?: number;
  /**
   * 写入方式。Agents 建议始终填充，供其它监听方使用。
   * VarUpdate 通知等级为 debug 时对非空值输出 trace（可与 Agents 对账）。
   */
  writeMode?: WriteMode;
}

/** agents:step_retry */
interface StepRetryPayload {
  agentId: string;
  agentLabel: string;
  stepIndex: number;
  /** 当前重试次数（第几次重试） */
  retryCount: number;
  /** 最大重试次数 */
  maxRetries: number;
  /** 重试原因 */
  reason: RetryReason;
}

/** agents:step_failed */
interface StepFailedPayload {
  agentId: string;
  agentLabel: string;
  stepIndex: number;
  /** 人类可读的错误描述 */
  error: string;
  /** 失败类型 */
  failureType: FailureType;
}

/** agents:pipeline_ended */
interface PipelineEndedPayload {
  /** 管道是否成功完成（所有步骤均通过） */
  success: boolean;
  /** 已完成的步骤数 */
  stepsCompleted: number;
  /** 总步骤数 */
  totalSteps: number;
  /** 是否被用户手动中止 */
  aborted: boolean;
}

// ═══════════════════════════════════════════════════════════
//  VarUpdate 广播的事件负载
// ═══════════════════════════════════════════════════════════

/** varupdate:initialized */
interface VarInitializedPayload {
  /** 初始化数据写入的消息索引 */
  messageIndex: number;
  /** 初始化后的完整变量状态 */
  data: Record<string, any>;
}

/** varupdate:updated */
interface VarUpdatedPayload {
  /** 触发更新的消息索引 */
  messageIndex: number;
  /** 成功执行的指令数 */
  appliedCount: number;
  /** 被丢弃的指令数 */
  discardedCount: number;
  /** 变更日志（路径 → "旧值 → 新值"） */
  log: Record<string, string>;
}

/** varupdate:update_failed */
interface VarUpdateFailedPayload {
  /** 触发更新的消息索引 */
  messageIndex: number;
  /** 失败原因描述 */
  reason: string;
  /** 被丢弃的指令数 */
  discardedCount: number;
}

/** varupdate:schema_ready */
interface SchemaReadyPayload {
  /** $defs 中定义的结构体名称列表 */
  defNames: string[];
}

// ═══════════════════════════════════════════════════════════
//  跨脚本协作事件负载
// ═══════════════════════════════════════════════════════════

/**
 * varupdate:retry_requested
 *
 * 由 Agents 广播，VarUpdate 监听。
 * 语义：当前步骤的输出不合格，即将重新生成。
 * VarUpdate 收到后将本次解析写入的变量全部撤销。
 */
interface RetryRequestedPayload {
  /** 需要回退变量的消息索引 */
  messageIndex: number;
}
```

#### 枚举类型

```typescript

  always: 1,
  notice: 2,
  error: 3,
```

#### 通配符匹配规则

```typescript
/**
 * 通配符匹配规则
 *
 * 使用场景：
 * - VarUpdate Schema 校验中的 $enum / $key_rule 等字符串匹配
 * - VarUpdate Schema 的 $enum / $key_rule 通配符匹配（面向用户 A-3）

 * - "开心***"    → 以"开心"开头
 * - "*开心*"     → 前后各恰好 1 字符
 * - "**民主主义" → 前面恰好 2 字符
```

#### 事件总线接口规范

```typescript
/**
 * 事件总线统一接口
 *
 * 两个脚本的事件总线模块（Agents 模块7 / VarUpdate 模块7）
 * 具有完全相同的接口签名和实现模式。
 */
interface EventBus {
  /** 广播事件 */
  emit(eventName: string, payload: any): Promise<void>;
  /**
   * 持续监听事件
   * @returns 包含 stop 方法的对象（与酒馆助手 eventOn 返回值一致）
   */
  on(eventName: string, handler: (payload: any) => void): { stop: () => void };
  /**
   * 一次性监听事件（触发后自动注销）
   * @returns 包含 stop 方法的对象（与酒馆助手 eventOnce 返回值一致）
   */
  once(eventName: string, handler: (payload: any) => void): { stop: () => void };
  /** 注销本脚本注册的所有事件监听器（脚本卸载时调用） */
  removeAll(): void;
}

/**
 * 底层实现：委托酒馆助手 API
 *
 * emit      → TavernHelper._bind._eventEmit.call(window, eventName, payload)
 * on        → TavernHelper._bind._eventOn.call(window, eventName, handler)
 * once      → TavernHelper._bind._eventOnce.call(window, eventName, handler)
 * removeAll → TavernHelper._bind._eventClearAll.call(window)
 *
 * 命名空间：
 * - Agents 事件使用 "agents:" 前缀
 * - VarUpdate 事件使用 "varupdate:" 前缀
 */
```

#### VarUpdate 双通道消息处理

```typescript
/**
 * VarUpdate 通过两个独立通道接收待扫描正文，并监听宿主事件做编辑 / 滑动 / 变量回退。
 *
 * 通道 A：agents:message_complete（优先）
 *   → 从负载 content 读取正文；messageIndex 可选。
 *   → messageIndex 缺省：主控制器按当前 chat 最后一层执行写入或继承。
 *   → messageIndex 显式传入但非法（非非负整数，或类型非 number / string）：仍处理正文，楼层推断同缺省，并 notify.debug（category: msg）。
 *   → writeMode：不改变变量写入语义；通知等级为 debug 时对非空 writeMode 输出 trace（标题 message_complete），便于与 Agents 对账。
 *
 * 通道 B：酒馆原生 message_received（备用）
 *   → 从 chat[messageIndex].mes 读取；messageIndex 须为 number 或可解析为非负整数的 string（实现见 `sanitizeMessageIndexForWrite`）。
 *   → 管道运行中跳过本通道；另以短时间窗对照近期 message_complete，避免与通道 A 重复处理同一条。
 *   → 下标缺失或非法、或 chat[i] 无 mes：不扫描正文，notify.debug（category: evt），避免静默失败。
 *
 * 其它宿主事件（下标规则同上）：
 *   → message_edited：有效下标且存在 mes 时重跑正文管线。
 *   → message_swiped：有效下标时刷新该层 message 变量快照至调试与面板；非法下标同上 notify.debug（evt）。
 *   → varupdate:retry_requested：负载须含有效 messageIndex；否则 notify.debug（category: life）后忽略。
 *
 * 互斥判定（避免同一条消息被两通道重复处理）：
 *
 * 主方案（事件标志位）：
 *   VarUpdate 维护内部标志（pipeline_started / pipeline_ended 置位）
 *   - 通道 B 触发时若管道活跃 → 跳过
 *
 * 备用方案（时间窗口）：
 *   通道 B 触发时若距上一次 message_complete 不足约定毫秒数 → 跳过（覆盖标志位未及时更新的极端情况）。
 */
```

#### 宏注册

### 4.6 错误类型

```typescript
/**
 * 所有自定义错误的基类
 * 附带 context 字段用于结构化错误信息

class ScriptError extends Error {
  constructor(
    message: string,
    public readonly context: Record<string, any> = {}
  ) {
    super(message);
    this.name = this.constructor.name;
  }
}

/** 格式解析错误（模块 1） */
class FormatParseError extends ScriptError {
  jsonError?: string;
  tomlError?: string;
  yamlError?: string;

class SchemaCompileError extends ScriptError {
  /** 所有编译错误（不因第一个就中断） */
  errors: Array<{
    path: string;
    type: string;
    message: string;
  }>;
}

/** Patch 解析错误（模块 3） */
class PatchParseError extends ScriptError {
  /** 原始文本（用于调试） */
  rawText: string;
```
